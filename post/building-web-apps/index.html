Building web apps

Mon, Jan 4, 2021 2-minute read

For the last three years I have been working in the backend of the web landscape by building backend services in Kotlin (and a bit of Java) using Spring Boot. In the climate I work in, motivation gets to you. I want to build my own ideas into prototype applications. But how am I going to get there? Ride along on the journey how I am using VueJS to build my ideas.

As a backend engineer I do not have that much experience in frontend applications so it is an adventure for me. What I often see is a split up application where there is a frontend application and a backend service (or more). I firstly preferred a setup where I had one framework where I could build it all. That means you look at something like Laravel, Rails and such. This first step was a random choice. I went for Rails. Why? Because of the marketing on social media by @dhh.

The CLI for it is amazing as it scaffolds it all for you. Creating a To Do app was easy, but it felt not right for me. You get a ton of directories containing a lot of different files, I do not want to see all that mess. Also, the language is just not for me. So, now we move on.
